# Using `script_args.py`

`src/script_args.py` is auto-generated by the UIBuilder from the `.uibproj`. It
defines a parsed `args` namespace organised by tab. The macro reads arguments
through it â€” never roll your own argparse parser in a Macro Automation Studio macro.

## Import

```python
from src.script_args import args
```

## Access pattern

```python
args.<tab_slug>.<arg_key>
```

Tab names are slugified to Python identifiers (`"My Settings"` â†’ `my_settings`).
The arg key is exactly what was set in the UIBuilder inspector.

### Example .uibproj â†’ access mapping

For a project with:

- Tab "General" containing `username` (text-input), `dry_run` (checkbox)
- Tab "Advanced" containing `threads` (number-input), `log_level` (combobox with values info/debug/warning)

The script reads:

```python
from src.script_args import args

print(args.general.username)         # str
print(args.general.dry_run)          # bool
print(args.advanced.threads)         # int (or float if step is fractional)
print(args.advanced.log_level)       # str â€” one of "info", "debug", "warning"
```

## Type-by-widget

| Widget kind | `args.<tab>.<key>` type |
| --- | --- |
| `text-input` | `str` |
| `text-area` | `str` |
| `number-input` | `int` if `step` is integer, else `float` |
| `slider` | `int` if `step` is integer, else `float` |
| `checkbox` | `bool` |
| `combobox` | `str` (one of the configured option values) |
| `radio-group` | `str` (one of the configured option values) |
| `file-picker` | `str` (absolute path) |
| `directory-picker` | `str` (absolute path) |
| `date-picker` | `str` (format from the widget, e.g. `"YYYY-MM-DD"`) |
| `time-picker` | `str` (format from the widget) |

## Required vs default

- An arg flagged `required` in the UIBuilder fails parsing if not provided. The
  user can't click Run without filling it in, so the script can assume it's
  populated.
- Optional args fall back to the widget's `defaultValue`. `None` is never the
  value â€” strings default to `""`, numbers to whatever the widget specifies.

## Empty / missing

If the user creates an arg widget but leaves the `key` blank, the UIBuilder skips
it entirely â€” neither `ui.xml` nor `script_args.py` will mention it. Validation
catches this on save.

If an arg appears in `script_args.py` but the user wants to run the script
manually (outside Macro Automation Studio), they pass it via CLI:

```
python src/app.py --username alice --threads 4 --dry_run
```

## Don't mutate

`args` is a parsed `SimpleNamespace`. Treat it as read-only â€” anything in `args`
is what the user picked in the runner dialog. Use locals or other state for
values you compute.
