# Macro project structure

A macro project is the directory the user creates from the Macro Automation Studio
desktop app. Every macro shares the same layout.

## Layout

```
my-macro/
├── my-macro.uibproj          # UIBuilder argument-form source (JSON)
├── my-macro.uibrt            # Runtime dashboard source (JSON, optional)
├── ui.xml                    # Generated by UIBuilder on save — do not edit
├── requirements.txt          # Optional third-party Python deps (mas is pre-installed)
└── src/
    ├── app.py                # Entry point — Macro Automation Studio runs this
    └── script_args.py        # Generated by UIBuilder from .uibproj — do not edit
```

## Entry point

`src/app.py` is the only file Macro Automation Studio runs. It's executed with the
CLI flags built from the form, and `mas.client._rpc_call` reaches a per-execution
RPC server bound to the device's emulator port.

A minimal `src/app.py`:

```python
import mas
from src.script_args import args


def main() -> None:
    print(f"Running with input: {args.general.input_path}")


if __name__ == "__main__":
    main()
```

Everything else (helpers, classes, sub-modules) can live in `src/` next to `app.py`
and be imported normally — the working directory is the project root.

## `requirements.txt`

The runtime venv comes with `mas` pre-installed (the host app sets that up). Use
`requirements.txt` only for additional third-party dependencies the script needs,
e.g.:

```
requests>=2.31
pillow>=10.0
```

Don't list `mas` here — it's provided.

## Generated files

- **`ui.xml`** — UIBuilder writes this on save. The desktop app reads it to render
  the runner dialog. Don't hand-edit; UIBuilder will overwrite.
- **`src/script_args.py`** — UIBuilder writes this on save. It contains the
  argparse parser and a `args` namespace organized by tab. Don't hand-edit.

## How the script reads args

```python
from src.script_args import args

# Each tab in the .uibproj becomes an attribute group
print(args.general.input_path)        # tab "General", arg key "input_path"
print(args.advanced.threads)          # tab "Advanced", arg key "threads"
print(args.flags.verbose)             # tab "Flags", arg key "verbose" (bool)
```

The tab name is slugified (`"My Settings"` → `my_settings`). The key is exactly
what was set in the UIBuilder Inspector. See `using-script-args.md`.

## Image Library

If the macro uses `find_object`, the user uploads template images via the desktop
app's Image Library. Each image gets an integer ID. The script declares them via
`mas.images({"login_btn": 123, ...})`. The IDs are project-specific and must be
provided by the user — the script doesn't generate them.

See `mas-images.md` for usage.
